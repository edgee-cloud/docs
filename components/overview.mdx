#### SDK Gateway

Edgee’s SDK Gateway components allow you to proxy and optimize **JavaScript SDKs** from analytics and marketing tools through your own domain.
These gateways improve resilience against ad blockers, improve performance, and ensure better data collection, even under strict browser policies.

These components securely inject JavaScript SDKs into your pages through your main domain, proxying all SDK calls through edge-optimized paths. 
This architecture makes the SDKs highly resilient against browser restrictions (like Safari's ITP) and ad blockers, while maintaining full JavaScript functionality. 

While still leveraging JavaScript, these gateways boost SDK reliability and performance by handling critical operations at the edge. 
They're particularly effective for platforms like Meta, Google, and Piano that support hybrid or server-side approaches.

#### Endpoint

Endpoint components allow developers to deploy lightweight HTTP microservices directly at the edge without standing up new backend infrastructure.
They're perfect for things like IP-based geolocation or storing small key-value pairs used at runtime.

When configuring an endpoint component, you'll need to specify a URL path pattern that determines when the component is activated. 
For example, you might configure the geolocation component to respond to requests at `/apigeo`. 
This gives you full control over your API routing while keeping the implementation simple and declarative.

Fast, serverless, and deployed in seconds - just configure your desired path and the component handles the rest.
